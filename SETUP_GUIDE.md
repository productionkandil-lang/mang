# دليل تجهيز Firebase — خطوة بخطوة

الخطوات دي لازم تتعمل مرة واحدة بس، وبحساب Google بتاعك (مش أنا اللي أقدر أعملها، محتاجة تسجيل دخول بحسابك).

## 1) إنشاء المشروع
1. افتح https://console.firebase.google.com
2. Add project → اكتب اسم زي `gm-marine` → تقدر تعطّل Google Analytics (مش لازم) → Create project

## 2) تفعيل تسجيل الدخول بالإيميل
1. من القائمة الجانبية: **Build → Authentication**
2. Get started → تبويب **Sign-in method**
3. فعّل **Email/Password**

## 3) إنشاء قاعدة البيانات
1. **Build → Firestore Database → Create database**
2. اختار **Start in production mode**
3. اختار أقرب منطقة (مثلاً `eur3` أو `europe-west3`) → Enable

## 4) تركيب قواعد الحماية (أهم خطوة)
1. من Firestore Database → تبويب **Rules**
2. امسح اللي موجود، والصق بدله محتوى ملف **firestore.rules** اللي مبعوتلك في نفس الملف المضغوط
3. اضغط **Publish**

## 5) تسجيل الموقع كـ Web App وأخذ الإعدادات
1. من ⚙️ بجانب "Project Overview" → **Project settings**
2. لتحت في "Your apps" → اضغط أيقونة **</>）** (Web)
3. سمّي التطبيق أي اسم → Register app
4. هيديك كائن `firebaseConfig` فيه قيم زي apiKey و projectId
5. افتح ملف **firebase-config.js** في الموقع، واستبدل القيم اللي فيها `PASTE_...` بالقيم دي بالظبط

## 6) إنشاء أول حساب أدمن
لسه مفيش حد يقدر يدخل الموقع لحد ما تعمل أول مستخدم يدويًا:
1. **Authentication → Users → Add user**
2. حط إيميلك الحقيقي وباسورد قوي → Add user
3. انسخ الـ **User UID** اللي ظهر جنب اسمه
4. روح **Firestore Database → Data → Start collection**
5. اسم الـ Collection: `users`
6. Document ID: الصق الـ UID اللي نسخته (مش auto-ID)
7. ضيف الحقول دي بالظبط:
   - `name` (string) → اسمك
   - `role` (string) → `admin`
   - `email` (string) → نفس الإيميل اللي سجلت بيه
8. Save

## 7) السماح لدومين الموقع بتاعك
1. **Authentication → Settings → Authorized domains**
2. ضيف الدومين اللي هترفع عليه الموقع (مثلاً `yoursite.netlify.app`)
   (localhost موجود افتراضيًا لو حبيت تجرب على جهازك الأول)

## 8) جرّب
1. افتح `login.html` بالإيميل والباسورد اللي عملتهم في خطوة 6
2. من صفحة "المستخدمين" ضيف باقي الناس (مدخل بيانات / مطّلع فلوس) بإيميلاتهم الحقيقية

---

## حالة الترحيل الحالية (إيه اتنقل فعليًا للسحابة وإيه لسه محلي)

| الصفحة | الحالة |
|---|---|
| تسجيل الدخول / المستخدمين | ✅ حقيقي بالكامل (Firebase Auth + Firestore) |
| الموظفين | ✅ حقيقي بالكامل — المرتب وبدل المواصلات في مستند منفصل ماله صلاحية قراءة أصلًا لمدخل البيانات |
| باقي الصفحات (أنظمة العمل، رفع الملف، بيانات الشهر، السلف، الإجازات، مواعيد تاريخية، عربية الوردية، تقفيل الشهر، الشهور السابقة) | 🔶 الدخول والصلاحيات حقيقية، لكن البيانات نفسها لسه بتتخزن محليًا على كل جهاز (زي الأول) — يعني كل جهاز/متصفح له نسخة بياناته الخاصة لحد ما ننقلها هي كمان لنفس نظام Firestore |

قولّي لما تخلص الخطوات دي وتتأكد إن تسجيل الدخول وصفحة الموظفين شغالين، وهكمل نقل باقي الصفحات بنفس الطريقة.
